# MCP-сервер для интеграции с IDE

Платформа предоставляет MCP-сервер (Model Context Protocol), который позволяет разработчикам запрашивать проиндексированную кодовую базу прямо из Claude Code, VS Code и других MCP-совместимых инструментов. Это сокращает расход токенов за счёт использования заранее проиндексированного кода вместо повторного чтения файлов.

## Предварительные требования

- Активная лицензия AI Pro ([см. руководство по активации](./ai-pro-license.md))
- Хотя бы один проиндексированный репозиторий ([см. руководство по индексации](./ai-repository-indexing.md))
- Подключение к LLM для инструмента `ask_about_code` ([см. настройку LLM](./ai-llm-connection.md))

## Шаг 1. Создание API-ключа

MCP-клиенты аутентифицируются через API-ключи (не через cookie браузера).

### Через UI настроек

1. Откройте **Settings > API Keys**
2. Нажмите **Create API Key**
3. Введите имя (например, «Claude Code»)
4. При необходимости установите срок действия
5. **Скопируйте ключ сразу** — он показывается только один раз

### Через API

```bash
curl -X POST http://localhost:3001/profile/api-keys \
  -H "Content-Type: application/json" \
  -b "auth_token=<jwt>" \
  -d '{ "name": "Claude Code" }'
```

## Шаг 2. Подключение из IDE

### Claude Code / Claude Desktop

Добавьте в конфигурацию MCP (`settings.json` или `claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "qa-platform": {
      "url": "http://your-server:3001/mcp/sse",
      "headers": {
        "Authorization": "Bearer qarp_ваш_api_ключ"
      }
    }
  }
}
```

## Доступные инструменты

### `search_code`

Семантический поиск по проиндексированному коду. Возвращает релевантные фрагменты с путями к файлам и оценками релевантности.

### `ask_about_code`

Вопрос-ответ на основе RAG. Извлекает контекст кода и генерирует ответ с помощью LLM.

### `get_file_context`

Получение проиндексированного контекста для конкретного файла или символа.

### `list_indexed_files`

Просмотр всех проиндексированных файлов с возможностью фильтрации по префиксу пути.

## Безопасность

- API-ключи хешируются SHA-256 перед сохранением — исходный ключ не хранится
- Ключи могут иметь срок действия
- Каждый ключ привязан к создавшему его пользователю
- `lastUsedAt` обновляется при каждом использовании
- Отозванные ключи немедленно отклоняются

## Управление ключами

```bash
# Список ключей
curl http://localhost:3001/profile/api-keys -b "auth_token=<jwt>"

# Отзыв ключа
curl -X DELETE http://localhost:3001/profile/api-keys/1 -b "auth_token=<jwt>"
```

## Устранение неполадок

| Проблема | Причина | Решение |
|----------|---------|---------|
| «Authentication failed» | Невалидный или просроченный ключ | Создайте новый ключ в Settings > API Keys |
| Инструменты недоступны | Сервер не отвечает | Проверьте URL и работу платформы |
| Пустые результаты поиска | Код не проиндексирован | Сначала запустите индексацию |
