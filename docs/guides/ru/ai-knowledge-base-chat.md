# База знаний: Чат по коду

Чат по базе знаний позволяет задавать вопросы о вашем проиндексированном коде в диалоговом интерфейсе. Он объединяет гибридный пайплайн поиска платформы (BM25 + векторное сходство + граф символов) с вашей LLM для предоставления контекстно-зависимых ответов с инлайн-ссылками на код.

## Предварительные требования

- Активная лицензия AI Pro ([см. руководство по активации](./ai-pro-license.md))
- Токен функции `retrieval` в лицензии
- Хотя бы один проиндексированный репозиторий ([см. руководство по индексации](./ai-repository-indexing.md))
- Настроенное подключение к LLM ([см. настройку LLM](./ai-llm-connection.md))

## Доступ к Code Chat

1. Откройте боковую панель и нажмите **Code Chat** (доступно при выбранном проекте)
2. Нажмите **New Chat** для начала разговора
3. Введите вопрос и нажмите **Ctrl+Enter** для отправки

## Как это работает

При отправке сообщения платформа:

1. **Извлекает** релевантные фрагменты кода через гибридный пайплайн поиска (топ-10 результатов)
2. **Формирует** системный промпт с контекстом кода (пути к файлам + сниппеты)
3. **Включает** историю диалога (последние 20 сообщений) для непрерывности
4. **Стримит** ответ LLM в реальном времени через SSE (Server-Sent Events)
5. **Сохраняет** ответ ассистента со ссылками на код для будущего доступа

## Возможности

### Потоковая передача в реальном времени

Ответы стримятся токен за токеном по мере генерации LLM. Вы можете остановить генерацию в любой момент кнопкой **Stop**.

### Ссылки на код

Каждый ответ ассистента включает сворачиваемые карточки со ссылками:
- Путь к файлу из проиндексированного репозитория
- Оценка релевантности (в процентах)
- Фрагмент кода, использованный как контекст

### Управление диалогами

- **Автозаголовок**: первое сообщение автоматически становится заголовком диалога
- **Несколько диалогов**: переключайтесь между диалогами в боковой панели
- **Удаление**: удаляйте ненужные диалоги
- **Привязка к проекту**: диалоги привязаны к активному проекту

## API-эндпоинты

Все эндпоинты требуют JWT-аутентификации (cookie `auth_token`).

### Создание диалога

```bash
curl -X POST http://localhost:3001/chat/conversations \
  -H "Content-Type: application/json" \
  -b "auth_token=<jwt>" \
  -d '{ "projectId": 1, "title": "Вопросы по авторизации" }'
```

### Список диалогов

```bash
curl http://localhost:3001/chat/conversations?projectId=1 \
  -b "auth_token=<jwt>"
```

### Отправка сообщения (стриминг)

```bash
curl -X POST http://localhost:3001/chat/conversations/1/messages/stream \
  -H "Content-Type: application/json" \
  -b "auth_token=<jwt>" \
  -d '{ "content": "Как работает middleware аутентификации?" }'
```

## Советы для лучших результатов

- **Будьте конкретны**: «Как Allure-импорт парсит лейблы?» работает лучше, чем «Как работает импорт?»
- **Ссылайтесь на файлы**: «Что делает `ai-analysis-retrieval.service.ts`?» помогает пайплайну поиска
- **Задавайте уточняющие вопросы**: история диалога включается в контекст
- **Сначала индексируйте**: убедитесь, что релевантный код проиндексирован

## Устранение неполадок

| Проблема | Причина | Решение |
|----------|---------|---------|
| «AI Analysis Pro is required» | Нет лицензии Pro или токена `retrieval` | Активируйте лицензию с токеном retrieval |
| Пустые или общие ответы | Код не проиндексирован или запрос слишком общий | Запустите индексацию, затем задайте более конкретный вопрос |
| Стриминг не работает | SSE заблокирован прокси | Проверьте, что прокси разрешает `text/event-stream` |
